mock agent나 scripted upstream이 아닌 **실제 사용자 100% 환경 UAT**를 진행한다. Codex CLI, 별도 localhost OAuth proxy/auth, 실제 ChatGPT 로그인 세션, `gpt-5.5`, `model_reasoning_effort=xhigh`, 실제 target repo fixture, visible/hidden acceptance gate를 함께 사용해 VibeLoop Harness의 다음 레벨 UAT를 검증한다.

핵심 변경점은 **API key 없이 OAuth/ChatGPT 로그인 기반 proxy 경로를 우선 경로로 승격**하는 것이다.

- 실제 UAT 실행용 문서와 runbook 작성

## 공식/OSS 참조 근거

- OpenAI Codex Auth 문서: Codex CLI/IDE는 ChatGPT sign-in과 API key sign-in을 모두 지원하며, ChatGPT sign-in은 access token을 CLI/IDE에 반환한다. 또한 custom model provider에서 `requires_openai_auth = true`를 사용하면 OpenAI 인증을 LLM proxy에 전달할 수 있고, 이 모드에서는 `env_key`를 무시한다.
- OpenAI Codex Models 문서: Codex에서 `gpt-5.5`가 권장 모델이며 ChatGPT 또는 API-key 인증에서 사용 가능하다.
- `EvanZhouDev/openai-oauth`: `npx openai-oauth`로 `http://127.0.0.1:10531/v1` OpenAI-compatible endpoint를 제공하고 API key 없이 Codex/ChatGPT OAuth cache로 `chatgpt.com/backend-api/codex/responses`에 연결하는 방식.
- `router-for-me/CLIProxyAPI`: 여러 CLI/Codex 계열을 OpenAI/Gemini/Claude/Codex compatible API로 감싸는 proxy reference.
- `BerriAI/litellm`: OpenAI-compatible gateway/proxy 서버 reference. 단, 본 UAT의 기본 경로는 API key gateway가 아니라 ChatGPT OAuth proxy다.

## 잔여 리스크 / 후속 과제

- 내부 OAuth forwarder는 `openai-oauth`와 동일 계열의 ChatGPT Codex backend 경로를 따르지만, upstream schema/헤더가 바뀌면 보정이 필요하다.
- 외부 OSS proxy는 매우 유용한 reference지만 unofficial이다. hosted service로 운영하지 않고 local/trusted 실험 경로로 제한해야 한다.
- Codex CLI의 인증/설정 인자가 버전별로 바뀌면 `-c` override 또는 custom provider 설정 보정이 필요할 수 있다.
- 100% 사용자 환경 UAT는 네트워크·계정 권한·모델 가용성에 영향을 받으므로 CI 기본 게이트가 아니라 수동/온디맨드 게이트로 유지한다.
