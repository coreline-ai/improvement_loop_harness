## Phase 2. [High] cancel 실효화

### 목표

cancel이 실행 중 loop를 실제로 중단하고, 그 후 결과가 terminal 상태를 덮지 못하게 한다.

### 구현 태스크

- [x] **signal 전파** (`queue.ts`): loop별 `AbortController`를 큐에 보관, `runner({ loop, task, signal })`로 전달. cancel 라우트는 active controller가 있으면 `abort()`하고, queued 등 아직 child가 없는 상태는 terminal `cancelled`로 선처리한다.
- [x] **terminal 덮어쓰기 가드** (`queue.ts`): 결과 기록 전 현재 loop status 재조회, 이미 `cancelled`/terminal이면 결과로 덮지 않고 `loop.result_ignored` 이벤트를 남긴다.
- [x] **child process kill 확인** (SDK cancel): `runCommand`가 `AbortSignal`을 받아 process group을 SIGTERM/SIGKILL 경로로 종료. agent adapter, baseline, test-on-base, command gate(host/container)가 같은 signal을 전달한다.

### 자체 테스트

- [x] 실행 중 cancel → runner 완료가 status를 accepted로 덮지 않음(가드 회귀).
- [x] cancel 후 loop가 cancelled로 남고 child가 정리됨(`runCommand('sleep 10')` abort 회귀 + API cancel signal 회귀).

### 완료 조건

- [x] cancel이 실행을 멈추고 terminal 상태가 보존된다.
