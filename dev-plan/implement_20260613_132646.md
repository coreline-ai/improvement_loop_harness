## 개발 목적

학습 루프 본체 구현 전에, 리뷰에서 저위험·고ROI로 식별된 **N4 metric 수집 신뢰화**와 **R5 git worktree prune 보강**만 독립적으로 구현한다.

핵심 목표는 다음 2개다.

1. baseline/candidate metric 수집이 stdout 정규식에만 의존하지 않도록 구조화 metric 채널을 우선 사용한다.
2. worktree remove fallback 후 stale git worktree metadata가 남지 않도록 `git worktree prune`을 무해하게 호출한다.

## 개발 범위

- `packages/eval-engine/src/baseline.ts`의 baseline metric 수집 신뢰화
- `packages/sdk/src/run.ts`의 candidate metric 수집 신뢰화
- gate command가 구조화 metric JSON 파일을 쓸 수 있는 안전한 artifact/env 계약 추가
- 구조화 metric 파일 우선, stdout 정규식은 legacy fallback으로만 유지
- 구조화 metric 파일 검증: JSON object, 허용 metric key, number finite 값만 인정
- stdout에 가짜 metric이 있어도 구조화 metric이 있으면 구조화 값만 사용
- `packages/workspace-runner/src/worktree.ts`의 `removeWorktree` fallback 후 `git worktree prune` 호출
- 관련 단위/e2e 회귀 테스트와 문서 반영

## 문서 기반 제약 사항

- 최종 accept 판정은 deterministic decision engine의 `ALL_PASS`만 허용한다.
- metric 수집 방식 변경은 evidence 입력 품질만 높이며 decision rule 순서를 바꾸지 않는다.
- 구조화 metric 파일도 untrusted gate output이므로 schema validation과 허용 key 필터를 통과한 값만 사용한다.
- stdout regex fallback은 호환성 목적이며 구조화 metric이 있으면 stdout metric을 덮어쓰지 못한다.
- prune 실패는 loop 실패로 만들지 않는다. cleanup hygiene 이벤트/로그만 남긴다.

## Phase 2. worktree prune 보강

### 목표

`git worktree remove --force` 실패로 rm fallback이 동작한 뒤에도 stale worktree metadata가 남지 않도록 정리한다.

### 구현 태스크

- [x] `removeWorktree` fallback 경로에서 `rm(ref.path)` 이후 `git worktree prune`을 호출한다
- [x] **락 재진입 금지(M1)**: `removeWorktree`는 이미 `withRepoLock(ref.lockPath)` 안에서 실행되므로, 같은 락을 다시 잡는 `pruneWorktrees()` helper를 **재사용하지 않는다**. 비재진입 파일 락이라 재획득 시 `LockTimeoutError`가 발생한다. 대신 이미 잡은 락 안에서 `safeGit(ref.repoPath, ['worktree','prune'], { timeoutMs })`를 **직접** 호출한다
- [x] prune 실패는 cleanup failure로 escalation하지 않고 무해 처리한다(`.catch(() => undefined)`)
- [x] prune은 정상 remove 성공 경로가 아니라 **rm fallback 경로에서만** 실행한다

### 자체 테스트

- [x] `safeGit worktree remove` 실패 시 rm fallback 후 prune 호출 확인
- [x] prune 실패가 `removeWorktree` 실패로 전파되지 않음
- [x] **M1 회귀**: fallback prune 경로가 `LockTimeoutError`를 던지지 않음(락 재진입 없음)
- [x] 정상 remove 경로에서는 prune을 호출하지 않음
- [x] lock 사용 경로 유지
