## 개발 목적

[implement_20260613_130927.md](./implement_20260613_130927.md) Phase 0 영역2 조사 결론(분리 구현)에 따라, **학습·메모리 루프(Learning Loop)**를 구현한다. 비전 §11("반복할수록 똑똑해진다")의 핵심으로, 루프가 끝날 때 구조화 자산을 남기고 다음 루프가 **참조 힌트로만** 사용한다.

핵심 목표는 다음 3개다.

1. 루프 종료 시 구조화 learning 레코드를 영속한다(원문 텍스트 미저장).
2. learning을 candidate **우선순위 가중**으로만 참조한다(판정/scope 불참, objective/프롬프트 주입 금지).
3. cross-loop failure ledger를 읽기 전용으로 조회한다.

## 개발 범위

- 루프 종료 시 `Learning` 레코드 영속(성공/실패 패턴·재사용 규칙, 구조화 요약)
- `Learning` 스키마 필드 보강(필요 시 Prisma migration)
- deterministic failure cluster key(`source` + `riskAreaHint` + `errorCode`)로 1차 군집화
- `ImprovementCandidate.priority`에 learning 기반 가중(참조 힌트만)
- cross-loop failure ledger 읽기 전용 조회 라우트/뷰(§11 포맷)

## 제외 범위

## 문서 기반 제약 사항

- 최종 accept 판정은 deterministic decision engine의 `ALL_PASS`만 허용한다.
- learning/memory는 읽기 참조·우선순위 힌트로만 사용하고 gate 통과/실패를 바꾸지 않는다.
- learning 자산·발견 입력은 untrusted — 원문 텍스트를 objective/프롬프트에 주입하지 않는다.
- learning 레코드에 hidden acceptance·token 원문을 저장하지 않는다.
