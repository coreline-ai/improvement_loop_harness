## 개발 목적

[implement_20260613_130927.md](./implement_20260613_130927.md) Phase 0 영역4 조사 결론(분리 구현)에 따라, **품질 수렴 종료 조건(convergence)**을 orchestrator 가드레일에 추가한다. 비전 §7의 "신규 실패 없음/평가자 임계 이상" 종료를 구현하되, 종료는 orchestrator **정지**일 뿐 accept가 아니다.

핵심 목표는 다음 2개다.

1. "처리 가능 candidate 없음 + 연속 무수확 N회"에서 graceful stop + 알림한다.
2. 종료 시 사람 종료 승인 게이트(§7.4)로 연결한다.

## 개발 범위

- `apps/server/src/orchestrator/guardrails.ts`의 `evaluateBeforeLoop`에 no-yield convergence 카운터 추가(기본값 안전값, 설정 가능)
- 종료 이벤트 `orchestrator.converged` 발행
- 종료 시 사람 종료 승인 큐 연결(기존 `createApproval`·`orchestrator.*` 경로 재사용)
- (선택) evaluator 임계 신호를 수렴 입력으로 사용하되 accept와 분리

## 문서 기반 제약 사항

- 최종 accept 판정은 deterministic decision engine의 `ALL_PASS`만 허용한다.
- 수렴 "종료"는 orchestrator 정지일 뿐 accept를 만들지 않는다.
- 기존 kill switch·일일 루프 예산·토큰 예산·연속 실패 차단기와 우선순위 충돌이 없어야 한다.
