이 문서는 **"live 실험됨"과 "제품으로 닫힘" 사이의 간극**을 메우는 검증 워크스트림의 작업 문서다. 현재 live 증거(실 GitHub repo + 실 Codex 수정 + draft PR)는 실재하나, 통제된 팀 테스트 repo(cart-total·sku)에 한정되고, adversary는 fixture 시나리오만 검증됐으며, 감사용 상세 report는 기본값으로 삭제된다. 이 셋을 actionable 검증 계획으로 닫는다.

> 과장 금지: 본 워크스트림은 **검증(validation)·증거(evidence) 중심**이다. 각 항목은 "PASS"를 증거 범위(매트릭스 셀/ledger 행/보존 경로)와 함께만 선언한다. 미신뢰 LLM 코드는 R1 격리에서만 실행. 자동 merge는 영구 금지.

## 개발 범위 (Phase)

- Phase 1 — **G3 감사 증거 영속화 (선행)**: live UAT가 eval/selection/quality/reverify report와 proxy auth stats를 기본 보존(또는 내구 artifact 저장소)하고, Run Ledger가 ephemeral 경로 대신 보존 경로를 가리키며, 감사 재현 절차를 문서화. **먼저 하는 이유: G1·G2의 증거 자체가 이 보존 위에 쌓인다.**
- Phase 2 — **G1 범용 repo 검증 매트릭스 (controlled)**: toolchain·규모·구조가 다른 대표 repo 셋에 대해 discover→run/improve→(필요시 PR)을 결정론 fixture builder로 검증하는 매트릭스 + 드라이버.
- Phase 3 — **G2 Live adversary lane**: 분리 컨텍스트 adversary가 _스스로_ 제안한 테스트를 정적 필터→M2 격리 confirm→M4 replay→freeze까지 **실 LLM·실 격리**로 돌리는 live UAT lane. (semantic runner 자체 구현은 `064636` 의존 — 본 Phase는 그 위의 live 검증.)
- Phase 4 — **G1 live 확장 (실 Codex)**: Phase 2 매트릭스 중 대표 1~2개를 실 Codex + 실 GitHub로 승격해 RU lane에 편입. broad full improvement는 과장 금지 선에서 controlled로 표기.
- Phase 5 — **종합 PASS 매트릭스·정직성 갱신**: 위 증거를 SKILL_REAL_USER_SCENARIO 매트릭스·Run Ledger·README "현재 상태"에 반영하되, 통제 범위를 명시.

- **Phase 완료마다 단일 커밋(`feat(prodval-phase-N): ...` / 검증은 `test(prodval-phase-N)`)** — 합쳐 커밋 금지.
- 증거는 항상 **보존 경로/ledger 행/매트릭스 셀**과 함께 보고한다. fixture PASS를 live PASS로, controlled PASS를 일반 배포 PASS로 부르지 않는다.
- live(실 Codex/GitHub) 검증은 비용·외부 의존이 있으므로, 기본 CI는 fixture/매트릭스, live는 명시 트리거(`uat:*`)로 분리한다.
- 미신뢰 코드 실행은 R1 격리·`network=none` 전제. 런타임 부재 시 fail-closed(skip이 아니라 명시 표기).
