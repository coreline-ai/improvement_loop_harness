`scripts/vibeloop-run.mjs`가 다음 순서로 CLI를 찾는다. 성공 경로에서는 **stderr 무출력**(UAT가 `stderr===''`를 단언).

1. `VIBELOOP_CLI` env (명시 override, node 스크립트 경로) — 항상 우선.
2. 모노레포 dev bin `../../../packages/cli/bin/vibeloop` (존재 시).
3. 스킬 내장 번들 `../vendor/vibeloop.mjs` (존재 시).
4. PATH의 `vibeloop` (전역 설치) — 최후.
5. 없으면 actionable 에러(해석 순서 + 해결법, exit 127).
