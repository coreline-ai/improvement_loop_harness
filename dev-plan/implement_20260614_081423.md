  - **정정**: 이전에 "환경상 구현 불가"라 한 것은 틀렸다. Docker CLI(29.2.0)가 있고 런타임은 colima다. R1은 구현 가능한 보안 워크스트림이다(프로젝트 plan도 별도 트랙으로 분리한 것이지 불가능이 아님).
  - **기반 구현 + 라이브 검증 완료**: `@vibeloop/shared` `runCommandInContainer`/`isContainerRuntimeAvailable`/`buildContainerInvocation` — `docker run --rm --network none -v <worktree>:/work` 격리 실행 primitive. 내부 명령은 env로 전달(argv 비노출). 순수 구성 테스트 4건(데몬 불필요) + docker-gated 라이브 테스트 4건.
  - **라이브 검증(2026-06-14, colima QEMU)**: VZ는 이 샌드박스에서 부팅 실패라 `brew install qemu` 후 `colima start --vm-type qemu`로 데몬을 띄워 **실제 컨테이너에서 검증**. 결과: echo 실행 ✓, `--network none` 네트워크 차단(NET_BLOCKED) ✓, /work 마운트 ✓, env 전달(argv 비노출) ✓. **선 테스트가 실제 버그를 잡음**: 초기 `sh -lc '"$VAR"'`는 "command not found"로 실패 → `sh -c 'sh -c "$VAR"'`(중첩)로 수정해야 연산자(&&/||/|/>) 포함 명령이 동작. 마운트는 런타임이 VM에 노출하는 경로($HOME 등)여야 함도 확인.
  - **gate-executor 배선 + 라이브 검증 완료(2026-06-14)**: `eval.yaml`에 opt-in `execution: { isolation: container, image, network }`. `executeCommandGate`가 isolation일 때 worktree와 구조화 metrics 디렉터리를 **동일 절대경로로 마운트**(cwd·`${WORKTREE_ROOT}`·`VIBELOOP_METRICS_FILE`가 컨테이너 안에서 그대로 해석)하고 `runCommandInContainer`로 실행. host PATH/env는 미전달(이미지 toolchain 사용). redact_gate_logs와 합성됨. **colima/QEMU 실측**: project gate가 리눅스 컨테이너 안에서 실행(uname=Linux) ✓, `--network none` 차단(NET_BLOCKED) ✓, 컨테이너가 쓴 `{"coverage_percent":77}`를 host가 metrics 파일로 읽음(마운트 동작) ✓, gate pass ✓. 기본(isolation 미설정) 경로는 host 실행 그대로 — unit 210·e2e 25 통과로 무변화 확인.
  - **P1 마무리 완료(2026-06-14)**: ① eval.schema.json `if/then`으로 `isolation:container`면 `image` 필수 → loadEvalConfig가 config error로 선검증(task-protocol 단위 테스트). ② docker-gated eval-engine 회귀 테스트 `gate-isolation.test.ts`(컨테이너 실행=Linux·`--network none` 차단·worktree 마운트·no-image→gate error) — colima/QEMU 라이브 통과. 부수: captureInMemory/error 분기의 gate log mkdir 누락 버그를 회귀 테스트가 잡아 수정(#8 redact 경로도 동시 hardening). image 없는 isolation은 gate error로 degrade.
  - **P2 진행(2026-06-14)**: M2 실행 + M4 replay substrate를 R1 위에 구현·라이브 검증.
    - **M2 실행 완료**: `confirmProposalUnderIsolation`(eval-engine) — 정적 필터를 통과한 adversary proposal의 (신뢰 불가) 테스트를 **R1 컨테이너 안에서만** 실행해 결정론적으로 확인(fail_to_pass=base fail∧candidate pass / pass_to_pass=candidate pass). 필터 탈락 proposal은 **절대 실행 안 함**. docker-gated 테스트 3건 colima/QEMU 통과.
    - **M4 substrate 완료**: `replayCorpusUnderIsolation`(eval-engine) — replay corpus를 R1 격리로 실행해 `replaySafe` 산출 → `decideShadowPromotion`에 연결(replaySafe true→promote, mismatch→replay_unsafe reject). docker-gated 테스트 2건 통과.
    - **남음(M4 전체)**: `RulepackRule`은 `{id,hash}` 추상 참조뿐 — 실행 가능한 rule 모델 + policy.lock/rulepack 계약(룰→corpus 매핑)은 **별도 설계 우선** 과제(R1 무관, 미구축 인프라). 본 substrate는 그 위의 격리 실행 기반.
